# Analysis 단계 가이드

## 목표
Spark에서 수집된 리서치 자료를 UI/UX/Feature 관점에서 분석하고 종합한다.

## 수행 작업
1. UI Lead — 시각적 디자인, 레이아웃, 컴포넌트 분석
2. UX Lead — 사용자 흐름, 인터랙션, 접근성 분석
3. Feature Lead — 기능 요구사항, 기술 제약, 구현 범위 분석
4. Coordinator — 크로스 리뷰 종합, 충돌 해결, 코드 매핑

## 주의사항
- 리서치 자료가 없으면 분석 불가 — 선택된 참고 자료 확인 필수
- Lead 간 분석이 상충하면 Coordinator가 명시적으로 해결
- 코드 매핑은 REUSE/MODIFY/NEW 분류로 정리
