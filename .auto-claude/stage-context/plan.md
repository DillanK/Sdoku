# Plan 단계 가이드

## 목표
Spec 요구사항을 아키텍처 설계와 실행 가능한 구현 계획으로 변환한다.
빌드 안전성을 유지하며 점진적으로 진행하는 순서를 설계한다.

## 아키텍처 설계 (architectureNotes 필수)
- 새 서비스/모델 구조, 기존 서비스 확장 방안
- 데이터 흐름 — 모델 변경, 상태 관리, 저장/로드 경로
- UI 설계 — 뷰 구조, 사용자 인터랙션 흐름
- 파일 변경 목록 — 신규 생성, 수정, 삭제 대상 확정
- AppState에 새 서비스 직접 추가 금지 — 별도 서비스로 분리 후 참조

## SubTask 설계 원칙
- 각 SubTask는 빌드가 깨지지 않는 단위로 설계
- Model/Service 먼저 → View 나중에 (의존성 방향)
- 테스트 가능한 로직은 테스트 SubTask 포함
- SubTask당 변경 파일 3~7개 권장
- SubTask는 너무 작게 쪼개는 것도 비효율적 (적정 크기 유지)

## 파일 수 기반 maxTurns 가이드
- 1~3개 파일: maxTurns 15
- 4~6개 파일: maxTurns 25
- 7개+ 파일: maxTurns 40
