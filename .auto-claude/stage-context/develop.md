# Develop 단계 가이드

## 목표
Plan의 SubTask를 실제 코드로 구현한다.

## 실행 프로세스
1. 수정 대상 파일 전체 읽기 (수정 전 연관 소스 확인)
2. 테스트 먼저 작성 (가능한 경우)
3. 프로덕션 코드 구현
4. 빌드 확인
5. 테스트 실행

## 코드 작성 규칙
- UI 업데이트는 @MainActor에서
- CheckedContinuation은 반드시 한 번만 resume
- Task 클로저에서 [weak self] 캡처
- 주석은 한국어로 작성
- 요청하지 않은 리팩토링, 기능 추가 금지

## 엄격 준수 사항
- 요청하지 않은 리팩토링, 기능 추가, 코드 스타일 변경 **절대 금지**
- 기존 코드의 의도를 최대한 유지하면서 **최소한의 변경**으로 구현
