## Spec 핵심 결정
- 요약: 9x9 그리드 UI, 숫자 패드, 메모(Pencil Mark) 토글, 실행취소(Undo) 스택, 실시간 충돌 감지를 포함하는 스도쿠 게임 플레이 핵심 화면 구현
- 핵심 요구사항:
  - 9x9 스도쿠 그리드를 화면에 렌더링하며, 3x3 박스 경계선을 시각적으로 구분하여 표시해야 한다
  - 셀 선택 시 해당 셀을 강조 표시하고, 동일 행/열/박스의 셀도 연관 강조 표시해야 한다
  - 1~9 숫자 패드를 통해 선택된 셀에 숫자를 입력할 수 있어야 한다
  - 퍼즐 생성 시 고정된 초기값 셀은 수정 불가 상태로 표시해야 한다
  - 메모(Pencil Mark) 토글 버튼을 통해 일반 입력 모드와 메모 입력 모드를 전환할 수 있어야 한다
- 수용 기준:
  - 9x9 그리드가 정상적으로 렌더링되고 3x3 박스 구분선이 명확히 보인다
  - 고정 셀 탭 시 숫자 패드가 비활성화되거나 입력이 무시된다
  - 메모 모드 토글 후 숫자 입력 시 해당 셀에 작은 숫자(메모)로 표시된다
- 제외 범위: 퍼즐 생성 알고리즘 구현 (별도 Feature: 스도쿠 퍼즐 생성 엔진), 게임 결과 CoreData 저장 및 통계 처리 (별도 Feature: 홈 화면 및 게임 기록 저장), 홈 화면 UI 구현, 힌트(Hint) 기능, 자동 메모 채우기 기능, 테마/다크모드 커스터마이징, 게임 일시정지 및 타이머 표시 (명시적 요구사항 없음)
- 영향 범위: Sdoku/Views/GameView.swift (신규 생성), Sdoku/ViewModels/GameViewModel.swift (신규 생성), Sdoku/Models/GameState.swift (신규 생성 — 셀 상태, Undo 스택, 메모 데이터 포함), Sdoku/Models/SudokuBoard.swift (신규 생성 또는 퍼즐 생성 엔진 모델과 연동), Sdoku/Views/Components/SudokuGridView.swift (신규 생성), Sdoku/Views/Components/NumberPadView.swift (신규 생성), Sdoku/App/Navigation (홈 화면에서 게임 화면으로의 네비게이션 경로 연결)
