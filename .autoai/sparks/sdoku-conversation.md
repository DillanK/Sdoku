### 공통 UX 패턴

- **입력 방식**: 셀 선택 → 숫자 패드 탭 (표준), 연관 행/열/박스 하이라이트
- **메모(Pencil Mark)**: 후보 숫자 기입 기능은 사실상 필수 기능
- **Undo/Redo**: 무제한 실행취소 스택
- **에러 표시**: 실시간 충돌 감지 (같은 행/열/박스 중복 하이라이트)
- **다크 모드**: 장시간 플레이를 위한 테마 커스터마이징

### 난이도 설계 베스트 프랙티스

- **초기 힌트 수 기반**: 빈 셀이 많을수록 어려움 (최소 17개 힌트가 유일해 보장 하한)
- **풀이 기법 기반 등급화**: Easy(Naked Single만), Medium(Hidden Pair), Hard(X-Wing 등), Expert(Swordfish, 체이닝)
- 4단계 난이도(쉬움/보통/어려움/극악)는 업계 표준과 잘 부합함

## 📋 프로젝트 현황

### 현재 상태

- **iOS 26.2** 타겟, iPhone + iPad 지원
- **Spark 단계** — 아이디어 구체화 진행 중

### 기존 항목과의 관계
- 현재 등록된 다른 아이디어가 없으므로 **중복 항목 없음**
- 기존 CoreData `Item` 엔티티는 템플릿 코드로, 게임 모델로 교체 필요

## 💡 추천 관련 기능

1. **메모(Pencil Mark) 시스템** — 후보 숫자 기입은 스도쿠 앱의 핵심 기능으로, 난이도가 올라갈수록 필수적이며 사용자 만족도에 직접적 영향
2. **게임 통계 & 기록** — 난이도별 클리어 시간, 최고 기록, 완료 횟수 등을 추적하면 로컬 플레이에서도 리텐션 동기 부여 가능
3. **데일리 챌린지** — 매일 새로운 퍼즐 1개를 제공하는 패턴으로, 네트워크 없이도 날짜 기반 시드(seed)로 구현 가능하며 일일 플레이 습관 형성에 효과적

## Sdoku — 대상 사용자 및 화면 구성 정의

### 1️⃣ 대상 사용자와 차별점

**주 사용자**: 스도쿠 입문자~중급자를 중심으로, 모든 레벨을 아우르는 범용 앱을 지향합니다.

**차별점**:
- **광고 완전 제거** — Sudoku a Day의 성공 패턴을 따라, 광고 없는 순수 플레이 경험 제공
- **경량성 + 완전 오프라인** — OpenSudoku처럼 네트워크 의존 없이 언제 어디서나 즉시 플레이 가능
- **깔끔한 미니멀 UI** — Sudoku a Day의 5.0 평점 요인인 "심플함"에 집중 (복잡한 기능 배제)

**데이터 기반**: 벤치마킹에서 상위 앱들이 모두 로컬 전용이면서도 높은 평가를 받은 점은, **차별점이 사양이 아닌 경험**임을 시사합니다.

### 2️⃣ 화면 구성 및 흐름

**4-화면 구조**로 설계합니다:

| 화면 | 역할 | 포함 요소 |
|---|---|---|
| **홈 화면** | 게임 시작점 | 난이도 버튼 4개(쉬움/보통/어려움/극악), 통계 요약(완료 횟수/최고 기록) |
| **게임 플레이** | 핵심 UX | 9×9 그리드, 숫자 패드, 메모 토글 버튼, 실행취소/재시작 |
| **설정** | 테마/접근성 | 다크모드 토글, 음향 on/off |
| **통계 상세** | 진행률 추적 | 난이도별 클리어 시간, 완료 횟수 |

**진행 흐름**:  
`홈 → 난이도 선택 → 게임 플레이 → 완료/포기 → 홈`

**설계 근거**:
- 벤치마킹의 "필수 기능" (메모, 실행취소, 에러 표시)만 포함
- 게임 통계는 로컬 리텐션 동기부여 (홈 화면 요약으로 중량감 최소화)
- Sudoku Blox의 "일일 챌린지" 개념은 v1에서는 제외 (MVP 이후 고려)

## Sdoku — MVP 범위 결정 및 개발 계획

### 1️⃣ MVP 우선순위: **홈 + 게임 플레이만 v1 필수**

**결정**:  
- **v1.0 포함**: 홈 화면(난이도 선택) + 게임 플레이(그리드, 숫자 패드, 메모, 실행취소)

3. **시간 투자 효율** — 스도쿠 퍼즐 생성 + 유효성 검증 로직이 복잡하므로, 그곳에 역량 집중이 필수

| **테스트 & 버그 픽스** | ⭐⭐⭐ | 2~3일 |
