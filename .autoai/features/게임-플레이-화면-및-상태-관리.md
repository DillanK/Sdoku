### 요약
9x9 그리드 UI, 숫자 패드, 메모(Pencil Mark) 토글, 실행취소(Undo) 스택, 실시간 충돌 감지를 포함하는 스도쿠 게임 플레이 핵심 화면 구현

### 기능 요구사항
- 9x9 스도쿠 그리드를 화면에 렌더링하며, 3x3 박스 경계선을 시각적으로 구분하여 표시해야 한다
- 셀 선택 시 해당 셀을 강조 표시하고, 동일 행/열/박스의 셀도 연관 강조 표시해야 한다
- 1~9 숫자 패드를 통해 선택된 셀에 숫자를 입력할 수 있어야 한다
- 퍼즐 생성 시 고정된 초기값 셀은 수정 불가 상태로 표시해야 한다
- 메모(Pencil Mark) 토글 버튼을 통해 일반 입력 모드와 메모 입력 모드를 전환할 수 있어야 한다
- 메모 모드에서 입력된 숫자는 셀 내에 작은 크기로 최대 9개까지 표시되어야 한다
- 실행취소(Undo) 버튼을 통해 마지막 입력 동작을 순차적으로 되돌릴 수 있어야 한다
- 숫자 입력 시 동일 행, 열, 3x3 박스 내 중복 숫자를 실시간으로 감지하고 충돌 셀을 시각적으로 표시해야 한다
- 모든 셀이 올바르게 채워졌을 때 게임 완료 상태를 감지하고 완료 처리를 해야 한다
- 지우기(Erase) 기능을 통해 선택된 셀의 입력값 또는 메모를 삭제할 수 있어야 한다
- 퍼즐 생성 엔진으로부터 난이도에 맞는 퍼즐 데이터를 전달받아 초기 상태를 구성해야 한다

### 비기능 요구사항
- 셀 선택 및 숫자 입력은 16ms(60fps) 이내에 UI가 반응해야 한다
- 충돌 감지 로직은 입력 즉시(동기적으로) 실행되어 지연 없이 피드백을 제공해야 한다
- Undo 스택은 최소 100단계 이상의 동작을 저장할 수 있어야 한다
- iOS 16 이상을 지원해야 하며 SwiftUI로 구현해야 한다
- 메모리 사용량이 과도하게 증가하지 않도록 Undo 스택 상한선을 설정해야 한다
- 오프라인 환경에서 완전히 동작해야 하며 네트워크 의존성이 없어야 한다

### 인수 기준
- 9x9 그리드가 정상적으로 렌더링되고 3x3 박스 구분선이 명확히 보인다
- 고정 셀 탭 시 숫자 패드가 비활성화되거나 입력이 무시된다
- 메모 모드 토글 후 숫자 입력 시 해당 셀에 작은 숫자(메모)로 표시된다
- 동일 행/열/박스에 동일 숫자가 있을 때 관련 셀이 충돌 색상으로 즉시 표시된다
- Undo 버튼 탭 시 직전 입력이 취소되고 이전 상태로 복원된다
- 모든 셀을 정답으로 채우면 완료 상태가 감지되어 완료 UI가 나타난다
- 지우기 버튼 탭 시 선택 셀의 입력값(일반 또는 메모)이 삭제된다
- 퍼즐 초기 데이터가 주어졌을 때 초기값 셀이 고정 상태로 그리드에 정확히 배치된다
